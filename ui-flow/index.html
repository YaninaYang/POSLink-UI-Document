    UI Flow

    From the perspective of a UI App, details of handling the complete transaction as a unit is not a concerning factor. It however needs to treat a transaction as a series of interfaces. Moreover, developers of a UI App are free from the responsibility of looking at a series as it does not know what the next interface is going to be. So, handling a single interface, which is defined by a unique identifier (Intent Action) is the unit of execution for a UI App.

    For the UI App, the first interface of a transaction is no different than any other. It is only a series of self-sufficient, independent interfaces that are invoked bundled with the information necessary to create the interface. One thing to be noted here, an interface is only a visual and logical unit. And it needs to satisfy a few requirements:

    • Prepare an Android Activity for the end user to interact with
    • [EntryReceive Status Broadcasts from the BroadPOS App (if the specific interface requires so)
    • [EntrySubmit a Request Broadcast to submit the user-inputted data
    • [EntryReceive a Response Broadcast as Acknowledgement
    • [EntryClose itself if it receives the Disconnect Message

    This is to say that the POSLink UI API does not place any constraint on the specific implementation of an interface. The developers may choose to make use of Activity and Fragment libraries in their own way. As long as the app declares itself to the Android OS to consume the Activity Intent identified by actions, it is fine.

    From the structural perspective of the communication that takes place between an UI App and the BroadPOS App, handling a Transaction is like handling interfaces, and being prepared to close the UI App if it receives the Disconnect Message. If however, the UI App does not implement the final interface of a transaction, which may change dynamically, the UI App may not receive the Disconnect Message at all. In such a case, the last interface would be drawn by the internal UI module, and the internal UI module would receive the Disconnect Message. Same goes for all Response Broadcasts and Status Broadcasts. These will be discussed shorty.
